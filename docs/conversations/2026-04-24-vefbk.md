# Session du 2026-04-24 — VeFBK.sol

## Decisions prises

### Architecture VeFBK
- **Modele Curve veCRV** : lock $FBK pour une duree (1 semaine a 4 ans) → obtention de $veFBK
- **Decroissance lineaire** : veFBK = amount * (end - now) / MAX_LOCK_DURATION
- **Lock arrondi a la semaine** : comme Curve, pour des expirations propres
- **Non-transferable** : pas de fonction transfer — c'est un poids de vote, pas un token
- **Censure-resistance** : withdraw() n'a aucune condition hors expiration (pas de KYC)

### Fonctions implementees
- `createLock(amount, lockDuration)` — cree un lock, transfere $FBK vers le contrat
- `increaseAmount(additionalAmount)` — ajoute $FBK sans changer l'expiration
- `extendLock(newUnlockTime)` — prolonge la duree sans changer le montant
- `withdraw()` — retire $FBK apres expiration (aucune condition supplementaire)
- `balanceOf(user)` — solde veFBK actuel (decroit lineairement)
- `balanceOfAt(user, timestamp)` — snapshot de vote a un timestamp donne
- `remainingLockDuration(user)` — secondes restantes
- `isExpired(user)` — bool
